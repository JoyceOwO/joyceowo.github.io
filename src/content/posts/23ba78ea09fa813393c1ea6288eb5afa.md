## 引言

如果你每天都在用 PowerShell，卻還停在預設提示列，那畫面大概只能說能用，但談不上舒服。PowerShell 美化不只是外觀問題，它還會直接影響你辨識路徑、Git 狀態與操作節奏。這篇整理一個很實用的做法：使用 **oh-my-posh** 幫 PowerShell 套用主題，讓你的終端機看起來乾淨一點，工作時也順手一點。

---

## 🎯 目標說明

這篇要完成的事情很單純：

- 使用 oh-my-posh 美化 PowerShell
- 查詢目前可用主題
- 套用指定主題到 PowerShell
- 了解這段指令到底做了什麼
---

## 核心內容

### 步驟一：查詢 oh-my-posh 主題

- oh-my-posh 內建很多主題，可以先全部列出來
- 建議先挑一個順眼、資訊量適中的版本
- 第一次先不要想太多，先找一個你看了不煩的就行
```powershell
# 查詢所有可用主題
Get-PoshThemes
```

補充說明：

- `Get-PoshThemes` 會列出目前可使用的主題
- 每個主題本質上都是一份 JSON 設定檔
- 主題通常會控制路徑、Git 分支、執行狀態、時間等資訊要怎麼顯示
為什麼這樣做：

- 先看主題清單，再決定你要哪種風格
- 不然你直接套主題，多半是亂試一輪，最後還是回來重選
適用情境：

- 剛開始接觸 PowerShell 美化
- 想快速找現成主題，不想自己刻設定
可能限制：

- 主題很多，第一次看會有點眼花
- 有些主題若字型沒設好，圖示可能顯示異常
---

### 步驟二：套用 PowerShell 主題

- 選好主題後，就可以直接套用到目前的 PowerShell 工作階段
- 下面這段是最核心的指令
```powershell
# 初始化 oh-my-posh，並套用指定主題
oh-my-posh init pwsh --config "$env:POSH_THEMES_PATH\\jandedobbeleer.omp.json" | Invoke-Expression
```

補充說明：

- `init pwsh`：表示要為 PowerShell 產生初始化設定
- `-config`：指定主題檔位置
- `$env:POSH_THEMES_PATH`：代表主題資料夾路徑
- `Invoke-Expression`：把前面輸出的 PowerShell 內容直接執行
為什麼這樣做：

- oh-my-posh 不是直接改你的視窗外觀
- 它會先產生一段 PowerShell prompt 設定，再由 PowerShell 立刻執行
- 這樣你可以馬上看到主題效果，適合快速測試
背後原理：

- PowerShell 的提示列可以自訂
- oh-my-posh 的工作，就是幫你生成新的 prompt 樣式
- 主題 JSON 則決定哪些資訊會出現、要怎麼排版
適用情境：

- 想快速測試主題效果
- 想先看畫面，不急著寫進永久設定
可能限制：

- 只對目前這個 PowerShell 視窗生效
- 視窗關掉後就失效，除非另外寫進 `$PROFILE`
---

### 步驟三：切換其他 oh-my-posh 主題

- 想換主題不用重裝
- 直接改掉主題檔名就好
```powershell
# 切換為另一個主題
oh-my-posh init pwsh --config "$env:POSH_THEMES_PATH\\paradox.omp.json" | Invoke-Expression
```

補充說明：

- 你不一定要用 `jandedobbeleer`
- 只要主題檔存在，就可以直接切換
- 建議你一次只換一個，直接看畫面最準
主題簡單比較：

| 類型 | 主題 | 特點 |
| --- | --- | --- |
| 耐看穩定 | jandedobbeleer | 資訊完整，很多人拿來當起手式 |
| 簡潔風格 | minimal | 畫面乾淨，不會太花 |
| 資訊偏多 | paradox | 顯示內容豐富，適合喜歡細節的人 |

不適用情境：

- 你只想要極簡提示列，不想顯示太多資訊
- 公司環境有限制，不能安裝額外工具或設定字型

## ⚠️ 常見問題 / 踩坑

### 1. 套用後出現亂碼或方塊

原因：

- 多半不是 oh-my-posh 壞掉
- 通常是字型沒有支援 Nerd Font 圖示
解法：

- 安裝 Nerd Font
- 再到 Windows Terminal 或你使用的終端工具中切換字型
---

### 2. 關掉 PowerShell 後主題就消失

原因：

- 你現在只是套用到當前工作階段
- 沒有寫進 PowerShell 的個人設定檔
解法：

```powershell
# 開啟 PowerShell 個人設定檔
notepad $PROFILE
```

然後加入：

```powershell
# 每次開啟 PowerShell 時自動套用主題
oh-my-posh init pwsh --config "$env:POSH_THEMES_PATH\\jandedobbeleer.omp.json" | Invoke-Expression
```

---

### 3. 指定主題時出現找不到檔案

原因：

- 主題路徑不正確
- 或目前環境變數沒有正確設定
解法：

```powershell
# 查看主題資料夾路徑
$env:POSH_THEMES_PATH

# 確認主題檔是否存在
Test-Path "$env:POSH_THEMES_PATH\\jandedobbeleer.omp.json"
```
