## 🧠 引言（這坑，你可能也踩過）

你明明只是想 `git fetch` 或 `pull`

結果 Git 突然翻臉：

```plain text
error: You're on a case-insensitive filesystem...
```

白話一點就是：

> 你的遠端有「大小寫不同但名字一樣」的分支，但你本機（Windows）分不出來，所以直接炸掉。

這種情境常見於：

- Windows / Mac（不分大小寫）
- Linux teammate 推了大小寫不同的分支 😇
如果不處理，你會：

- fetch / pull 卡死
- 分支列表異常
- CI/CD 可能出事
---

## 🎯 目標

這篇要幫你做三件事：

1. 找出「大小寫衝突的分支」
1. 確認要保留哪個
1. 刪除多餘的分支
---

## 🔍 Step 1：找出大小寫衝突分支

PowerShell 指令：

```powershell
git ls-remote --heads origin |
ForEach-Object { ($_ -split "`t")[1] } |
Group-Object { $_.ToLower() } |
Where-Object { $_.Count -gt 1 } |
Select-Object -ExpandProperty Group
```

### 📌 原理說明

| 步驟 | 說明 |
| --- | --- |
| `git ls-remote` | 抓遠端所有 branch |
| `split` | 取 branch 名稱 |
| `ToLower()` | 全轉小寫 |
| `Group-Object` | 分組 |
| `Count > 1` | 找出重複 |

### 📌 範例輸出

```plain text
refs/heads/Feature/Login
refs/heads/feature/login
```

👉 這就是衝突來源

---

## ⚔️ Step 2：決定保留哪個

建議：

- 統一用 `feature/login`
- 或依團隊命名規範
👉 重點：只能留一個

---

## 💣 Step 3：刪除多餘分支

刪除遠端分支：

```bash
git push origin --delete Feature/Login
```

或：

```bash
git push origin --delete feature/login
```

---

## ⚠️ 常見踩坑

### 1️⃣ 本地 cache 沒清

```bash
git fetch --prune
```

---

### 2️⃣ CI/CD 還在用舊分支

記得檢查：

- pipeline
- webhook
- 部署腳本
---

### 3️⃣ reftable 解法（不建議）

Git 會提示：

```bash
git refs migrate --ref-format=reftable
```

👉 但實務上：

- 工具支援度不高
- 可能造成相容問題
👉 建議：直接清掉衝突分支

## 🧩 Bonus：快速檢查工具

```powershell
function Find-GitCaseConflict {
    git ls-remote --heads origin |
    ForEach-Object { ($_ -split "`t")[1] } |
    Group-Object { $_.ToLower() } |
    Where-Object { $_.Count -gt 1 } |
    Select-Object -ExpandProperty Group
}
```

使用方式：

```powershell
Find-GitCaseConflict
```
