## 結論

- 用 Hosts 直接指定「IP → 網址」，可繞過 DNS，快速驗證或修正連線問題
## 適合用在哪裡

- 測試環境 DNS 尚未設定完成
- DB / API 網址解析錯誤
- VPN / Private Link 導致 DNS 指錯 IP
- 想強制連到指定機器（除錯用）
## 流程步驟

### 1. 修改 Hosts 檔案

- 到路徑 `C:\\Windows\\System32\\drivers\\etc`
- 用「系統管理員」開啟 `hosts`
- 新增一行 IP 對應網址，直接覆蓋 DNS 解析
```plain text
10.10.0.1 test.database.windows.net
```

- 本機會優先使用這筆設定，不再查 DNS
### 2. 套用設定

- 儲存後不一定立即生效，需刷新 DNS 快取
```bash
ipconfig /flushdns
```

- 避免還在用舊的 DNS 結果
### 3. 驗證是否生效

- 用 `ping` 或 `nslookup` 確認解析結果
```bash
ping test.database.windows.net
```

- 若 IP 顯示為 `10.10.0.1`，代表設定成功
## 補充

- Hosts 只影響「本機」，不會影響其他人
- 若連線異常，優先檢查是否有舊 Hosts 殘留
- Azure / Private Endpoint 常會被 Hosts 覆蓋導致誤判
## 指令 / 範例整理

```plain text
# Hosts 設定

# 清除 DNS

ping test.database.windows.net
```

## 收尾

- Hosts 是最快的「強制導流工具」，但記得用完要清，別害未來的你
