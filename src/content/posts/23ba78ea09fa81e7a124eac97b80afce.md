## 為什麼要用 Robocopy？

在備份資料、同步資料夾、進行排程備份時，`Robocopy` 是 Windows 系統內建的強大工具。相比 `xcopy` 或 `copy`，它具備：

- 高容錯能力（失敗重試）
- 支援多執行緒加速
- 可完整保留 NTFS 權限與時間戳
- 適合處理大量檔案搬移與同步
---

## 🧭 目標說明

我們要做的是：

> 將來源資料夾的內容備份至目的地，過程中忽略特定資料夾與檔案，並將操作紀錄寫入 Log 檔案中。

---

## 🛠️ Robocopy 指令拆解

```plain text
cmd
複製編輯
robocopy "來源資料夾" "目的地資料夾" /mir /xj /MT:100 /xf *.scc *.bak ^
  /xd "packages" "bin" "obj" "node_modules" ".vs" "TempFile" "Data" "history" ^
  /maxage:7 > .\Log\%dd%.log


```

### 參數說明：

| 參數 | 說明 |
| --- | --- |
| `"來源資料夾"` | 要備份的來源路徑 |
| `"目的地資料夾"` | 備份目的地 |
| `/mir` | Mirror 模式，會同步來源與目的地，**包含刪除不存在的檔案** |
| `/xj` | 排除 Junction Points（避免循環遞迴或重複備份） |
| `/MT:100` | 使用 100 個執行緒平行處理，加快複製速度（上限 128） |
| `/xf *.scc *.bak` | 排除指定副檔名的檔案 |
| `/xd ...` | 排除特定資料夾 |
| `/maxage:7` | 只處理近 7 天內修改的檔案（依修改日期） |
| `> .\Log\%dd%.log` | 將輸出紀錄寫入指定 log 檔案（`%dd%` 可由 bat 產生當日日期） |

---

## 🧩 實作補充建議

1. **Log 資料夾先建立**：
```plain text
cmd
複製編輯
if not exist ".\Log" mkdir ".\Log"
```

1. **%dd% 自動取日期**（bat 中寫法）：
```plain text
cmd
複製編輯
set dd=%DATE:~0,4%%DATE:~5,2%%DATE:~8,2%
```

1. **搭配 Windows Task Scheduler 定時執行**：適合日常備份排程。

## ✅ 總結

這篇介紹了一個典型的 Robocopy 指令範例，幫你：

- 同步資料夾（鏡像）
- 排除不必要的檔案與資料夾
- 記錄操作 log
- 限制同步最近 7 天的檔案
- 用多執行緒加速複製效率
這樣的指令組合非常適合在 CI/CD 前、每日備份、部署前備份等情境中使用。
