## Conclusion

This script uses the `DatabaseName` parameter to control the export scope: if a value is provided, only the specified DB is exported; if it's empty, all user databases under the entire Azure SQL Server are automatically exported.

## Use Cases

-   Want to test if a single DB export works correctly
-   Want to back up all user databases on an entire Azure SQL Server at once
-   Want to make the backup process a repeatable PowerShell script
-   Want to retain each export result and error message for future tracing

## Steps

### 1. Define Parameters and Login Information

First, prepare the Azure resource name, Storage location, SQL Admin account, and add the `DatabaseName` parameter. This parameter is the key focus of this article: an empty value means all DBs, while a value means a single specified DB. Then, enter the SQL Admin password and retrieve the Storage Account Key, which will be used later for bacpac export.

```powershell
# ===== Basic Parameters =====
$ResourceGroupName   = "your-rg"
$ServerName          = "your-sql-server"
$DatabaseName        = ""   # Empty = All DBs; Value = Specific single DB
$StorageAccountName  = "yourstorageacct"
$ContainerName       = "sql-backup"
$SqlAdminLogin       = "your_sql_admin"

# A prompt will appear; please enter your Azure SQL admin password
$SqlAdminPassword = Read-Host "Enter Azure SQL admin password" -AsSecureString

# Get Storage Account Key
$StorageKey = (Get-AzStorageAccountKey `
    -ResourceGroupName $ResourceGroupName `
    -Name $StorageAccountName)[0].Value
```

### 2. Determine Which Databases to Process Based on DatabaseName

This step only processes the list, not the export. If `DatabaseName` is empty, it retrieves all databases under the logical server, excluding `master`; if it has a value, it creates a list for only that single DB. This allows the subsequent export process to be shared, avoiding the need to write two separate scripts.

```powershell
if ([string]::IsNullOrWhiteSpace($DatabaseName)) {
    # Get databases under this logical server, excluding master
    $databases = Get-AzSqlDatabase `
        -ResourceGroupName $ResourceGroupName `
        -ServerName $ServerName |
        Where-Object { $_.DatabaseName -ne "master" }
}
else {
    # If a DB is specified, create a list for only that single DB
    $databases = @(
        [PSCustomObject]@{
            DatabaseName = $DatabaseName
        }
    )
}
```

### 3. Export Each bacpac to Blob Storage

Each DB is appended with a timestamp to prevent overwriting files with the same name. Then, `New-AzSqlDatabaseExport` is called to submit an export request, writing the bacpac to the specified Blob Container. This is the core and most practical part.

```powershell
# Create result log
$results = @()

foreach ($db in $databases) {
    $CurrentDatabaseName = $db.DatabaseName
    $TimeStamp = Get-Date -Format "yyyyMMdd-HHmmss"
    $BacpacFileName = "$CurrentDatabaseName-$TimeStamp.bacpac"
    $StorageUri = "<https://$StorageAccountName.blob.core.windows.net/$ContainerName/$BacpacFileName>"

    Write-Host "Starting export for $CurrentDatabaseName ..."

    try {
        $exportRequest = New-AzSqlDatabaseExport `
            -ResourceGroupName $ResourceGroupName `
            -ServerName $ServerName `
            -DatabaseName $CurrentDatabaseName `
            -StorageKeyType "StorageAccessKey" `
            -StorageKey $StorageKey `
            -StorageUri $StorageUri `
            -AdministratorLogin $SqlAdminLogin `
            -AdministratorLoginPassword $SqlAdminPassword
```

### 4. Continuously Query Status and Organize Results

Export is not a synchronous operation, so `Get-AzSqlDatabaseImportExportStatus` is used to poll the status. Once completed, the status, output location, and error message for each DB are collected into a results table, and finally output uniformly. This prevents a situation where you only see one DB stuck without knowing the status of others.

```powershell
        # Poll export status
        do {
            Start-Sleep -Seconds 15
            $status = Get-AzSqlDatabaseImportExportStatus `
                -OperationStatusLink $exportRequest.OperationStatusLink

            Write-Host "$CurrentDatabaseName => $($status.Status)"
        } while ($status.Status -eq "InProgress")

        $results += [PSCustomObject]@{
            DatabaseName = $CurrentDatabaseName
            Status       = $status.Status
            StorageUri   = $StorageUri
            ErrorMessage = $status.ErrorMessage
        }
    }
    catch {
        $results += [PSCustomObject]@{
            DatabaseName = $CurrentDatabaseName
            Status       = "Failed"
            StorageUri   = $StorageUri
            ErrorMessage = $_.Exception.Message
        }
    }
}

$results | Format-Table -AutoSize
```

## Additional Notes

-   It's recommended to exclude `master` directly, as it usually has no practical value for bacpac backup.
-   It's recommended to use `$CurrentDatabaseName` and not overwrite the original `$DatabaseName` parameter within the `foreach` loop, to avoid confusion later.
-   If you encounter issues like unsupported schema, cross-DB references, or external objects, export failures are often due to bacpac limitations, not PowerShell syntax errors.
