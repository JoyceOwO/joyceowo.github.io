## Conclusion

- By directly mapping "IP → Hostname" in Hosts, you can bypass DNS to quickly verify or fix connection issues.
## Where It's Useful

- DNS not yet configured in test environments
- Incorrect DB / API hostname resolution
- VPN / Private Link causing DNS to point to the wrong IP
- Want to force a connection to a specific machine (for debugging)
## Steps

### 1. Modify the Hosts File

- Navigate to `C:\Windows\System32\drivers\etc`
- Open `hosts` with "Administrator" privileges
- Add a line mapping an IP to a hostname, directly overriding DNS resolution
```plain text
10.10.0.1 test.database.windows.net
```

- The local machine will prioritize this setting and no longer query DNS.
### 2. Apply Settings

- After saving, it might not take effect immediately; you need to flush the DNS cache.
```bash
ipconfig /flushdns
```

- This prevents using old DNS results.
### 3. Verify Effectiveness

- Use `ping` or `nslookup` to confirm the resolution result.
```bash
ping test.database.windows.net
```

- If the IP displayed is `10.10.0.1`, the setting is successful.
## Additional Notes

- Hosts only affects the "local machine" and will not impact others.
- If connection issues occur, prioritize checking for any lingering old Hosts entries.
- Azure / Private Endpoint configurations can often be overridden by Hosts, leading to misdiagnosis.
