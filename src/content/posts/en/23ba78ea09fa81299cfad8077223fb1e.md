## Conclusion

- By directly mapping "IP → Hostname" in Hosts, you can bypass DNS to quickly verify or fix connection issues.
## Where It's Useful

- DNS is not yet configured in the test environment
- Incorrect DB / API hostname resolution
- VPN / Private Link causes DNS to resolve to the wrong IP
- To force a connection to a specific machine (for debugging)
## Steps

### 1. Modify the Hosts File

- Navigate to `C:\Windows\System32\drivers\etc`
- Open `hosts` with "Administrator" privileges
- Add a line mapping IP to hostname, directly overriding DNS resolution
```plain text
10.10.0.1 test.database.windows.net
```

- The local machine will prioritize this setting and no longer query DNS.
### 2. Apply Settings

- It may not take effect immediately after saving; you need to flush the DNS cache.
```bash
ipconfig /flushdns
```

- To prevent using old DNS results.
### 3. Verify Effectiveness

- Use `ping` or `nslookup` to confirm the resolution result.
```bash
ping test.database.windows.net
```

- If the IP displayed is `10.10.0.1`, it means the setting is successful.
## Additional Notes

- Hosts only affects the "local machine", not others.
- If connection issues occur, prioritize checking for residual old Hosts entries.
- Azure / Private Endpoint configurations can often be overridden by Hosts, leading to misdiagnosis.
