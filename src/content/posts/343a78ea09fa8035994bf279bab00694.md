## 結論

- OLE DB Driver 19 可能導致 Token 登入失敗，直接降版到 18.7.5 可解
## 適合用在哪裡

- 使用 OLE DB 連線 SQL Server（含 Azure SQL）
- 使用 Token / AAD 驗證
- 發生 `<token-identified principal>` 登入錯誤
- SSIS / Linked Server / 外部系統整合
## 流程步驟

### 1. 確認錯誤訊息

- 出現以下錯誤：
  - `Login failed for user '<token-identified principal>'`
  - `The server is not currently configured to accept this token`
- 通常發生在：
  - 使用 OLE DB Driver 19
  - 搭配 Azure AD / Token 驗證
- 這不是 SQL 權限問題，而是 Driver 相容性問題
### 2. 移除 OLE DB Driver 19 (選擇性)

- 到「控制台 → 程式與功能」
- 找到：
  - Microsoft OLE DB Driver 19 for SQL Server
- 進行解除安裝
- 避免多版本混用（可能導致連線仍抓到 19）
### 3. 安裝指定版本 18.7.5

- 下載舊版 Driver（官方）：
  - [https://learn.microsoft.com/en-us/sql/connect/oledb/release-notes-for-oledb-driver-for-sql-server?view=sql-server-ver17#1875](https://learn.microsoft.com/en-us/sql/connect/oledb/release-notes-for-oledb-driver-for-sql-server?view=sql-server-ver17#1875)
- 安裝：
  - Microsoft OLE DB Driver 18.7.5
- 安裝完成後重新啟動服務（如 IIS / SSIS / App）
## 補充

- Driver 19 在某些 Token 驗證場景有相容性問題（特別是舊系統）
- 若使用 Linked Server，也會影響連線測試
- 不建議混裝 18 / 19，避免 DLL 載入混亂
## 收尾

- 有些錯不是你權限問題，是版本在搞你，降版反而最快解
